# Security Policy

## 🔐 Supported Versions

Only the latest stable release of **FightPlanner** is actively supported with security updates.

| Version | Supported |
| ------- | --------- |
| 4.X.X and 3.X.X  | ✅ Yes |
| Under of 4.X.X and 3.X.X | ❌ No |

We strongly recommend always using the most recent version to benefit from the latest security fixes.

---

## 🚨 Reporting a Vulnerability

If you discover a security vulnerability in **FightPlanner**, please **do not open a public GitHub issue**.

You can report it securely using **one of the following methods**:

### GitHub Security Advisories (preferred)
- Use GitHub’s **“Report a vulnerability”** feature on this repository  
- This allows private discussion and responsible disclosure

### Email
- **Email:** firexdf@shtrdchs.fr or firexdf.contact.pro@gmail.com

When reporting a vulnerability, please include:
- A clear description of the issue
- Steps to reproduce
- Potential impact
- Any proof of concept (if available)

---

## ⏱️ Response Time

We aim to:
- Acknowledge security reports within **24 hours**
- Provide a fix or mitigation as quickly as possible, depending on severity

---

## Responsible Disclosure

We kindly ask security researchers to:
- Allow reasonable time to address the issue before public disclosure
- Avoid exploiting vulnerabilities beyond what is necessary for testing
- Respect user data and privacy at all times

We sincerely appreciate your efforts to help keep **FightPlanner** secure

---

## 🔄 Policy Updates

This security policy may be updated at any time. Please check this document periodically for changes.
